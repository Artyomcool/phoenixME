Technology Compatibility Kits

As a given API goes through the JCP standardization process, three components are developed in parallel:

  • A Java specification request (JSR) defines an API.

  • A reference implementation (RI) demonstrates the specification.

  • A technology compatibility kit (TCK) validates the RI against the JSR with a collection of test programs.

At this time, the specific TCK tests for CDC-related JSRs are not available as open source. However, the systems used to build TCKs are available through the testME Project and these systems can be used by OSS developers to build test suites for their applications and libraries.

JT Harness and ME Framework

As part of the testME Project, Sun has open sourced some of the test technologies that it uses to build TCKs for the Java ME platform:

  • The JT Harness is a general-purpose testing harness.

  • The ME Framework facilitates the testing of small devices with the JT Harness.

So for a given JSR, a TCK is a bundle of tests that is hosted on and driven by the JT Harness and that interact with test devices through the ME Framework.
